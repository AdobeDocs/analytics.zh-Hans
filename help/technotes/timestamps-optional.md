# 使用可选时间戳

了解有关使用可选时间戳设置的好处和限制。

可选时间戳是所有新报表包的默认设置。

* 在同一全局报表包中混合加盖时间戳和未加盖时间戳的数据。
* 将加盖时间戳的数据从移动应用程序发送到全局报表包。
* 升级应用程序以使用时间戳，无需创建新的报表包。

>[!NOTE]“可选时间戳”是从模板生成的所有新报表包的默认设置。从现有报表包复制而来的新报表包将继承原始报表包的设置。

有关其 [他设置信息](https://marketing.adobe.com/resources/help/zh_CN/reference/timestamp-optional.html) ，请参阅可选时间戳。

## 可选时间戳：集成加盖时间戳和未加盖时间戳的数据 {#section_BF17CB593044462B993FD0D28EA56518}

使用可选时间戳功能，您可以将未加盖时间戳的数据与加盖时间戳的数据组合在一起，而不会造成数据丢失。 具有从移动设备生成的时间戳的离线数据可以与来自网页的实时、未加盖时间戳的数据组合，或使用客户端时间戳调用与来自任何平台的数据集成。

* **时间戳数据**。 可使用以下客户端时间戳变量捕获客户端时间戳数据，并直接随设备数据一起发送：在网页上使用 Javascript，或在移动应用程序中使用移动 SDK 调用 ([!DNL offlineEnabled=true])。
* **非时间戳数据**。 当数据到达收集服务器时，Adobe会为报表包中未加盖时间戳的数据设置时间戳。


报表包可以具有以下时间戳设置之一：

* 禁用时间戳（支持设置 visitorID）
* 必须提供时间戳（不支持设置 visitorID）
* 可选时间戳（支持设置 visitorID，但不适用于带有时间戳的点击）

## 关于时间戳可选功能 {#section_63B2FA9A2AB24B3993E84D2C2B4BF2CE}

时间戳可选允许您在包含或不包含客户端时间戳的多个报表包之间集成和报告。 使用可选时间戳，您可以更新应用程序以使用时间戳，同时仍使用上一个应用程序中未加时间戳的数据。

| 在以前的版本中…… | 此外…… |
|--- |--- |
| 无法将加盖时间戳的数据发送到未加盖时间戳的全局报表包。 因此，发送自离线设备的点击数据将在添加到未加盖时间戳的报表包时被丢弃。<br/><br/>因此，发送自离线数据的点击数据将在添加到未加盖时间戳的报表包时被丢弃。 | 在更新应用程序以收集和使用时间戳时，要求您应用新的报表包。<br/>您在更新应用程序以使用时间戳时，无法保存到现有的报表包或集成现有的数据。 |

**通过可选时间戳**，您可以将实时网站上未加盖时间戳的数据与移动设备中的离线数据集成，或将您未加盖时间戳的应用程序更新为加盖时间戳的应用程序。

## 将数据整合到全局报表包中 {#section_5BE3BDF56007402BB1F5C3144D5FE1E0}

将数据整合到全局报表包中可通过多种方式完成，包括多包标记、VISTA 规则，以及从离线来源导入批量文件。

>[!IMPORTANT]请仔细规划每个组件数据集的设计，从而在全局报表包中实现整合。

## 应用时间戳时的最佳实践 {#section_9436394E5D7E4F8A8B369B6D11BB2B2B}

以下是将加盖时间戳的数据与未加盖时间戳的数据集成时需要了解的最佳实践以及一些要求和限制。

* 通常，给定访客或访问的时间戳必须按正确的时间顺序到达Adobe。

   无序数据可以包括来自离线数据收集的延迟到达数据和延迟到达的点击，或离线移动设备上的不同步时钟。 无序数据可能会对时间计算（如停留时间值）、归因（eVar持久性）、访问次数／访问次数和路径报告产生负面影响。

* 不建议在设置 [s.visitorID时使用时间戳](https://marketing.adobe.com/resources/help/en_US/sc/implement/visid_custom.html) 。 它可能导致数据出现无序。

* 由打开Web浏览器（未加盖时间戳的实时数据）的应用程序（加盖时间戳的脱机数据）组成的混合应用程序不应使用时间戳。 这会导致会话的报告不准确。

   此外，混合应用程序不应设置访客ID。
