# 数据保留策略

Adobe Analytics收集的数据将在特定时间段内保留。Adobe保留这些数据的时间因合同而异，并且在组织的数据保留策略中列出。此策略适用于数据本身，这意味着它影响所有Analytics报告功能(Analysis Workspace、报表API等)。

**Adobe Analytics的默认数据保留策略为25个月。** 您组织的保留策略可能不同，具体取决于合同。

保留的数据基于历史数据的当前日期和日期/时间。点击时记录的日期/时间可能与Adobe收到的点击日期/时间不同。

## 调整默认数据保留时间段

如果您要减少或延长默认数据保留期，请与贵组织的客户经理联系。

* 减少默认数据保留期是没有费用的。
* 将数据保留延长到默认保留时间超过25个月需要购买扩展，这些扩展可按每年的增量提供。最多可购买八个扩展，总共可购买10个月(默认保留期限为2年，外加8年购买)。

## 数据保留和GDPR

Adobe作为您的数据处理者，必须采取适当措施来协助其客户执行访问、删除和其他来自个人的请求。“运用适当、安全和及时的删除策略”是履行此项职责的重要部分。GDPR 适用于以欧盟公民为行销对象或处理欧盟公民相关信息的所有客户。因此，GDPR 是一个全球范围的法规变动。

## 删除数据

数据超出您的数据保留策略后，Adobe保留删除该政策的权利，而不会选择恢复。您必须确保所有要保留的数据都属于组织的数据保留策略。

## 常见问题解答

**如何确定组织的数据保留期限？**

作为数据管理者，您的公司可以在负责数据保留决策的组织中识别利益相关者(例如您的营销、分析和隐私团队)。您的组织最好了解Adobe Analytics保留数据的时间段。

**如何计算数据保留窗口？**

数据保留策略定义了一个滚动的数据保留窗口时间，在这段时间内，您可以查看和报告已完成的数据。数据保留开始日期取决于当前日期减去数据保留期。数据保留结束日期由当前日期决定。如果数据的时间戳介于开始日期和结束日期之间，则该数据涵盖在此数据保留窗口时间内。

**我可以在删除之前请求一份数据副本吗？**

能。Adobe可以提供原始数据、点击级别数据的历史数据转储。See [Data Feeds](../export/analytics-data-feed/c-getstarted/data-feed-overview.md) in the Export user guide for more information. 如果您的数据导出要求超出UI可以提供的范围，请与单位的客户经理联系。可以进行特殊安排；成本可能有所不同。

**Adobe何时删除数据？**

在计划删除您的数据的特定时间，与贵组织的客户经理联系。数据通常会按月删除。
