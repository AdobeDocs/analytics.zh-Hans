# Adobe Analytics使用的域

某些防火墙配置会阻止Adobe Analytics作为其产品界面所依赖的域。 您可以使用此域列表来更改组织的网络设置，以允许从组织内部访问产品。 Adobe建议允许这些域通过贵组织的防火墙以获得最佳体验。

| 技术 | 域 |
| --- | --- |
| Adobe Analytics 域 | `adobe.com`, `adobe.net`, `adobe.io` |
| Adobe Analytics 旧域 | `omniture.com` |
| Amazon AWS | `aaui-879784980514.s3.us-east-2.amazonaws.com` |
| Amazon CloudFront | `d30ln29764hddd.cloudfront.net` |
| Gainsight | `esp.aptrinsic.com`、`esp-m.aptrinsic.com` |
| LaunchDarkly | `app.launchdarkly.com` |
| ® Azure Blob Storage | `awaascicdprodva7.blob.core.windows.net` |
| ® AZURE CDN | `aauicdnva7.azureedge.net` |

## Adobe CX Enterprise域

除了上述域外， Adobe CX Enterprise还依靠多个域进行数据收集和导出报告。 有关此域列表，请参阅[CX Enterprise使用的域](https://experienceleague.adobe.com/en/docs/core-services/interface/data-collection/domains)。
