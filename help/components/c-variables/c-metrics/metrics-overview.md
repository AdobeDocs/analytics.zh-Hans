# 量度快速参考

列出 Adobe Analytics 中的标准量度。

>[!NOTE]
>
>以下未列出的任何度量（事件）都是自 [定义度量](../../../components/c-variables/c-metrics/metrics-custom.md#concept_F44638FC95A44B06AEBA3A6F9D008D27) （自定义事件）。

>[!IMPORTANT]
>
>Analysis Workspace no longer distinguished between Traffic and Conversion metrics. Hence, the metric type is only relevant to tolls such as Reports &amp; Analytics, Web Services 1.4, and Report Builder.)

| 量度名称 | 描述 | 类型 |
|--- |--- |---|
| 平均页面深度 | 显示访问中触发每个值的平均深度。此量度用于确定一次访问中访客到达给定页面或 prop 值的深度。平均页面深度可用于任何启用了路径分析的变量。 | 流量 |
| 页面平均逗留时间 | 表示某次访问中的页面平均逗留时间。 | 流量 |
| 网站平均逗留时间 | 表示某次访问中的网站平均逗留时间。 | 流量 |
| 跳出率 | 显示包含单次点击的访问次数的百分比。跳出率使用跳出次数量度并计算为：“跳出次数”除以“登录次数”。 | 转化 |
| 跳出次数 | 只包含一次服务器调用的访问。例如，如果访客没有与页面进行需要向 Adobe 发送数据的交互操作（如单击链接或启动视频），则一次页面访问就是一次跳出。如果访问中收到的点击次数超过一次，那么将不会计为一次跳出。 | 转化 |
| 促销活动点进次数 | 点进次数表示给定促销活动的跟踪代码被传递到报表中的次数。当访客点击某个附属活动链接（该链接已采用其中的一个跟踪代码进行标记）时，访客将被引入到您的登录页面，同时，跟踪代码也被捕获到 s.campaign 中。该数据会被发送到报表，并记录一次点进。 | 转化 |
| 购物车加货 | 商品被添加到购物车的次数。该值来自于 scAdd 事件。 | 转化 |
| 购物车打开 | 客户添加第一件商品而打开购物车的次数。发生在第一次向购物车添加商品时。该值来自 scOpen 事件。 | 转化 |
| 购物车减货 | 从购物车中移除商品的次数。此值来自 scRemove 事件。 | 转化 |
| 购物车 | 打开或初始化新购物车的次数。 | 转化 |
| 购物车查看 | 客户查看购物车内容的次数。 | 转化 |
| 结账 | 客户在购买过程中到达结账阶段时发生的事件。结账通常是购物完成之前的最后一个步骤，而且往往需要客户输入个人信息（例如送货和帐单信息）。您可以控制您网站上被确定为结账的事件。此值来自 scCheckout 事件。 | 转化 |
| 点进次数 | 点进次数表示给定促销活动的跟踪代码被传递到报表中的次数。当访客点击某个附属活动链接（该链接已采用其中的一个跟踪代码进行标记）时，访客将被引入到您的登录页面，同时，跟踪代码也被捕获到 s.campaign 中。该数据会被发送到报表，并记录一次点进。 | 转化 |
| 客户（新客户、回头客户、忠诚客户） | 客户忠诚度报表的类别：新客户：购物 0 次的客户。回头客户：购物 1 次的客户。忠诚客户：购物 1 次以上的客户。 | 流量 |
| 每日回访 | 显示在指定的日期访问您网站的次数超过一次的访客数量。一日定义为过去 24 小时。 | 流量 |
| 登录 | 登录次数表示作为访问中第一个值所捕获到的给定值的次数。登录在每次访问中只能发生一次。但是，如果未定义变量，则它不一定是首次点击。 | 流量 |
| 退出 | 作为访问中最后一个值所捕获到的给定值的次数。退出在每次访问中只能发生一次。 | 流量 |
| 实例 | 针对变量设置某个值的次数。所有点击类型都会计算实例数，但是在后续点击上针对变量记录值时将不计算实例数量，因为这会一直存留。 | 转化 |
| 移动设备查看次数 | 通过移动设备访问时，页面的查看次数或维度的设置次数。仅限 Ad Hoc Analysis。我们建议应用“来自移动设备的访问次数”区段，而不使用移动设备查看次数量度。 | 转化 |
| 新参与 | 新参与是一个营销渠道报表量度，可计算某个渠道带来的新访客。此量度还可计算最近 30 天中未到访您网站的访客。“新参与”是在每次访问开始设置的 eVar（原始分配）。首次联系渠道也可以是新参与，具体取决于访客参与到期设置。 | 转化 |
| 发生次数 | 捕捉特定值的次数加上保留给定值的页面查看数量。也就是说，发生次数是页面查看次数和页面事件数之和。发生次数只在 Ad Hoc Analysis 中可用。 | 在Reports &amp; Analytics、Web Services 1.4或Report builder中不可用 |
| 订购 | 选定时间段内网站上的订购数目。您可通过其他量度划分各个时间段，以显示此时间范围内促成最多订购的项目（如产品或促销活动）。 | 转化 |
| 页面深度 | 用户到达网站中特定页面所经过的平均点击数。 | 流量 |
| 页面事件 | 页面事件包括来自非标准图像请求的图像请求数据。非标准图像请求的来源是下载链接、退出链接和自定义链接跟踪。 | 流量 |
| 页面查看次数 | 每个已发送的服务器调用都计为一次页面查看。此量度代表页面查看的总实例数。TrackLink 调用不被计为页面查看次数，因此不会作为“页面查看”量度的增量。 | 转化 |
| 路径查看次数 | 路径查看量度基于路径数据，将为所有接受永久 Cookie 的用户跟踪该数据。术语“路径查看”用于表示在所显示的路径范围内查看某个页面的次数；换句话说，此量度报告了指定页面在选定路径中被查看的次数。此量度可用于路径报表中。路径查看次数显示了查看特定顺序的页面的次数。 | 流量 |
| 产品查看次数 | 设置产品查看的实例。查看“产品详细信息”页面时发生。此值来自 prodView 事件。 | 转化 |
| 重新载入 | 当同一页面名称连续被加载两次时计数。这通常表明页面已被刷新。请注意，同一访问中访问同一页面两次不会被计为重新加载，除非连续两次进行访问。 | 流量 |
| 回访 | 显示访问次数大于 1 的访问量。回访包括不带 Cookie 的访客。 | 转化 |
| 收入 | 收入是在购买事件中捕获的，它被定义为订购每件产品的合计美元金额。此值来自购买事件。 | 转化 |
| 搜索 | 搜索不是默认量度，它始终为自定义量度。它是搜索引擎和关键词的推荐默认量度。此量度代表点进操作的实例，可显示与特定引擎或关键词关联的页面。搜索量度数据可以被追溯报告到数据集的开始位置。 | 转化 |
| 单次存取 | 单次存取由网站的访问量（包含一个唯一的页面名称值）定义。如果用户进入您的网站并点击跟踪链接、触发了事件（例如视频查看），或重新载入页面，则访问仍会被视为一次单次存取。只要 pageName 变量的值没有更改，就可以发送任意次请求，并且访问仍然会被视为一次单次存取。 | 流量 |
| 逗留时间 | 作为一个量度，它可以报告访客在一个页面、网站上的逗留时间，或每次访问的逗留时间。 | 流量 |
| 合计 | 合计量度报告某个报告期间所有报表行项目的值。如果当前选择了一个过滤器，则总计信息可能表示过滤的总计信息而不是报表包的总计信息。如果未选择任何过滤器，则总计信息表示报表包的总计信息。 | 度量的总版本遵循基本度量的类型。 例如，“流量” `totalpageviews` 是“流量”，因 `pageviews` 为“流量”。 |
| 独特客户 | （每小时、每日、每周、每月、每季、每年）独特客户在该时间范围内只会被记录一次，而无论该访客返回购物多少次，都不会再次记录。独特访客在指定时段内首次访问时会被记录一次，然后在该时段过期之前都不会被再次记录。时段过期之后，独特访客会被再次记录。独特客户始终被算作独特访客，因为他们必须访问网站才能购物。 | 转化 |
| 独特访客 | 显示报告期间的独特访客总数（可以配置为每日、每周、每月、每季、每年）。 | 转化 |
| 件数 | 选定时间段内订购的总件数。由于您在每次订购中可以购买多件物品，因此“件数”是一项重要量度，可揭示总体存货的流动情况。 | 转化 |
| 访客 | 显示选定时间段（小时、日、周、月、季或年）内网站独特访客的人数。 | 转化 |
| 访问 | 表示一系列的页面查看操作。访问量度通常用在显示选定时段内用户会话数量的报表中。访问量度始终与某个时段关联，因此当同一访客返回您的站点时，您可以知道是否将其认为是新的访问。 | 转化 |
