# 管理虚拟报表包

通过虚拟报告套件管理器，管理员可以编辑、添加、标记、删除、重命名、批准、复制、导出和过滤虚拟报告套件。 虚拟报表包管理器对非管理员用户不可见。

**[!UICONTROL Analytics]** > **[!UICONTROL Components]** > **[!UICONTROL Virtual Report Suites]**

![](assets/vrs-manage.png)

>[!NOTE]在虚拟报表包管理器中，您只能看到自己的虚拟报表包。You have to click **[!UICONTROL Show All]** to see everyone else&#39;s.

| 任务 | 描述 |
|--- |--- |
| 添加 | 将您带到虚拟报表包构建器，在该生成器中可以创建新的虚拟报表包。 |
| 标记 | 所有用户都可以为区段创建标记，并将一个或多个标记应用于区段。 但是，您只能查看您拥有的区段的标记。 您应该创建哪些类型的标记？ 以下是一些有用标记的建议：<ul><li>基于团队名称的标记，如Social Marketing、Mobile Marketing</li><li>项目标记(分析标记)，如登录页面分析</li><li>类别标记：男士；地理</li><li>工作流标记：（特定业务单位）;已批准</li></ul> |
| 删除 | 如果您删除虚拟报表包，则应用了此虚拟报表包的计划报表和功能板将继续正常工作。该报表或功能板将继续使用已删除的虚拟报表包，直到您重新保存计划报表为止。编辑具有相同名称的虚拟报表包时，计划报表不会更新。<br>例如：假设您有两个名称相同，但父报表包不同的虚拟报表包：<br>您有一个书签，该书签引用 mainprod 报表包的虚拟报表包。然后，由于该虚拟报表包是重复项，您将其删除。书签将继续运行，并引用已删除VRS的定义。 如果更改其余VRS的定义，则应用于书签的VRS不会更改。 它使用旧的定义。 要解决此问题，请更新书签以引用新定义。 如果不确定书签、功能板或计划报表是否在使用已删除的 VRS，可以更改剩余 VRS 的名称，这样可以更加明确书签是否在使用剩余的 VRS。 |
| 重命名 | 与在报表包选择器中一样，在显示虚拟报表包的所有位置都会显示新名称。 |
| 批准／取消批准 | 批准虚拟报表包，使其成为“正式”或“规范”报表包。 您可以通过取消批准来撤消此过程。 |
| 复制 | 使用自己的新报表包ID创建一个不同的副本，但名称和定义相同。 |
| 导出到CSV | 将虚拟报告包定义导出为。csv文件。 |
| 过滤器 | 按标记、父报表包、所有者及其他过滤器（“显示所有”、“我的”、“收藏夹”和“已批准”）进行过滤。 |
