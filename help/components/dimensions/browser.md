# 浏览器

&#39;[!UICONTROL 浏览器]“维度报告发送点击的浏览器的名称和版本。 此维度有助于了解访客最常使用的浏览器。 测试网站的新版本时，您可以在此维度中的热门浏览器上运行这些测试，以便最大限度地做好质量控制工作。

## 使用数据填充此维度

此维度引用 Adobe 内部的一个查找表。查找值基于图像请求中的 `User-Agent` HTTP 标头。如果您使用 AppMeasurement 库（例如，通过 Adobe Experience Platform 中的标记），则此维度可开箱即用。

## 维度项目

维度项目包括使用的浏览器名称和版本。同一浏览器的不同版本是不同的维度项目。

某些维度项目包含 `"(unknown version)"`，而不是版本号。此维度项目会引用 Adobe 尚未添加到其查找表的最新浏览器版本。由于浏览器经常更新，因此给定浏览器的 `"(unknown version)"` 很常见，而且是临时的。Adobe 通常在月度维护版本发布期间更新查找表。

## 标签和定义中的更改

以下是对浏览器在报表中的显示方式所做的更改列表。 这些更改可能会影响您的报表或任何依赖这些值的逻辑，如区段。

### 从浏览器中删除公司

从版本100的Chrome、Edge和Firefox浏览器开始，公司名称将不再显示在浏览器之前。 如果用户具有基于浏览器名称的区段定义，则这可能会影响用户。 例如，包含定义“浏览器包含“Google”的区段将不再识别从版本110开始的Chrome浏览器。

示例：

Chrome版本109将显示为“Google Chrome 109”。
Chrome版本110将显示为“Chrome 109”。

### 消除Chrome中移动设备和非移动设备之间的区别

从Chrome 110开始，Chrome的移动版和非移动版将被标记为相同。 目前，移动版本和非移动版本分别进行标记。 重要的是，这会改变名称在没有“移动设备”的情况下的含义。 “Chrome 109”是非移动设备（即台式机）“Chrome 110”是移动设备，而非移动设备。 同样，如果您的浏览器名称中具有引用“mobile”的区段定义，则这些定义可能不再按预期工作。 您可以使用移动设备分段和划分来比较移动设备流量与非移动设备流量。

示例：

移动设备Chrome 109将显示为“Chrome移动设备109”。
非移动设备的Chrome 109将显示为“Chrome 109”。

移动设备Chrome 110将显示为“Chrome 110”。
非移动设备的Chrome 110将显示为“Chrome 110”。
