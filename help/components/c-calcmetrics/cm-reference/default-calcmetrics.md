# 默认计算量度

Adobe Analytics提供了各种计算指标以涵盖最常见的用例。 默认情况下，这些计算量度在Analysis Workspace中可用。

以下是Adobe提供的每个计算指标及其目标函数以及用于计算该函数的基础公式的列表：

>[!NOTE]
>
>除了本页中所述的默认计算量度之外，您还可以向报表包添加其他计算量度。
>
>您可以:
> * 为流媒体添加默认计算指标，如中所述 [计算量度](/https://experienceleague.adobe.com/docs/media-analytics/using/implementation/variables/calculated-metrics.html)
> * 从现有指标创建自定义计算指标，如中所述 [计算量度和高级计算（派生）量度](/help/components/c-calcmetrics/cm-overview.md).



| 计算量度名称 | 函数 | 公式 |
|---------|----------|---------|
| 跳出率 | 包含一次点击的访问次数与该页面上的访问次数之间的比率。 这有助于您了解哪些维度项目的跳出率最高，或查看一段时间内网站的累计总跳出率。 | 跳出率 |
| 收入/访客 | 网站每位访客产生的平均收入金额。 | <p>收入</p><p>/</p><p>独特访客</p> |
| 订购/访客 | 网站每位访客生成的平均订单数或交易数 | <p>默认订单</p><p>/</p><p>访客</p> |
| 收入/访问 | 单次访问网站产生的平均收入额。 | <p>收入</p><p>/</p><p>独特访问</p> |
| 收入/订单 | 网站上每个已完成的交易或订单所产生的平均收入金额。 | <p>收入</p><p>/</p><p>订购</p> |
| 订购/访问 | 导致完成交易的网站访问百分比。 | <p>订购</p><p>/</p><p>访问次数</p> |
| 页面查看/访问 | 用户在单次访问网站期间查看的平均页面数。 | <p>页面查看次数</p><p>/</p><p>访问</p> |
| 访问/访客 | 独特访客访问网站的平均次数。 | <p>访问次数</p><p>/</p><p>独特访客</p> |
| 重新载入/页面查看次数 | 导致重新加载或刷新页面的页面查看次数的百分比。 | <p>重新载入</p><p>/</p><p>页面查看次数</p> |
| 加权跳出率 | 函数 | if(访问次数>百分位数（访问次数），跳出，0) |
| 订购协助 | 渠道或来源对客户购买历程做出了贡献，但未导致最终购买的次数。 | <p>订购（参与率\|访问）</p><p>-</p><p>订单数</p> |
| 退出率 | 查看特定页面后离开网站的访客百分比。 | <p>退出</p><p>/</p><p>访问次数</p> |
| 登录率 | 在给定页面上进入网站的访客相对于网站上会话总数的百分比。 | <p>登入次数</p><p>/</p><p>访问次数</p> |
| 网站平均逗留时间 | 访客离开或导航离开之前在网站上逗留的平均时间。 | 网站平均逗留时间（秒） |
| 逗留时间/用户（状态） | 访客在网站上处于某个特定状态所花费的平均时间 | 每位访客逗留时间（秒）量度+移动应用程序区段 |
| 用户（移动设备） | 移动应用程序的用户总数 | 独特访客量度+移动应用程序用户区段 |
| 应用程序用户 | 移动应用程序的用户总数 | 独特访客量度+移动应用程序区段 |
| 状态视图 | 查看应用程序的不同状态或屏幕的次数 | 页面查看次数量度+移动应用程序区段 |
| 操作 | 在应用程序中执行的操作总数 | 自定义链接实例量度+具有操作区段 |
| 客户获取链接点击次数 | 用户点击旨在提升网站流量的链接的次数。 | 营销活动点进次数量度 |
| 页面周转率 | 一段内容生成的其他页面查看次数。 这可以帮助您确定哪些内容可推动更多参与。 | <p>页面查看次数</p><p>/</p><p>访问次数</p> |
| 转化率 | 执行所需操作（例如购买）的访客百分比。 | <p>订单数</p><p>/</p><p>访问次数</p> |
| 平均会话时长（移动设备） | 在一次会话期间，访客在网站上逗留的平均时间。 | 空白 |
| Experience CloudID覆盖 | 具有Experience CloudID的访客所占的百分比。 | <p>具有 Experience Cloud ID 的访客</p><p>/</p><p>独特访客</p> |
| 内容周转率 | 在网站上创建和发布新内容的速度以及它生成用户参与度的速度。 | <p>页面查看次数</p><p>/</p><p>访问次数</p> |
| ITP 2.1独特访客/独特访客 | 使用浏览器且受ITP 2.1 Cookie限制影响的独特访客百分比。 换言之，客户不使用CNAME实施。 （这适用于通过客户端JavaScript设置Cookie的客户。） | <p>独特访客量度+ ITP访客区段</p><p>/</p><p>独特访客</p> |
| 7天后回访的独特访客/独特访客 | 在7天或更长时间后回访的独特访客的百分比。 | <p>独特访客量度+7天后回访的用户区段</p><p>/</p><p>独特访客</p> |
| 页面查看次数/独特访客 | 网站每位独特访客查看的平均页面数。 | <p>页面查看次数</p><p>/</p><p>独特访客</p> |
| 访问 / 访客 | 独特访客对网站的平均访问次数。 | <p>访问次数</p><p>/</p><p>独特访客</p> |
| 预计独特访客(ITP 2.1) | <p>对于ITP访客（Safari浏览器中的用户），独特访客数除以2或更少。</p><p>这假定您使用客户端JavaScript（不使用CNAME实施）设置Cookie。 从ITP 2.1开始，使用客户端JavaScript设置Cookie的实施受到了影响。请参阅： [https://webkit.org/blog/8613/intelligent-tracking-prevention-2-1/](https://webkit.org/blog/8613/intelligent-tracking-prevention-2-1/)</p> | <p>独特访客量度+ ITP访客（ITP 2.1，非CNAME实施）区段</p><p>/</p><p>独特访客量度+非ITP访客（ITP 2.1，非CNAME实施）区段</p> |
| 页面查看次数/预计独特访客(ITP 2.1) | “预计独特访客”(ITP 2.1)的平均查看页面数。 | <p>独特访客量度+ ITP访客（ITP 2.1，非CNAME实施）区段</p><p>/</p><p>独特访客量度+非ITP访客（ITP 2.1，非CNAME实施）区段</p> |
