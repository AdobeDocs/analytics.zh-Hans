# “媒体播放逗留时间”面板

Media Analytics客户可以分析播放所用时间，以了解出现并发峰值或发生锐减的位置，从而对内容质量和查看者参与度提供有价值的分析，并有助于对容量或规模进行疑难解答或规划。

在Analysis Workspace中，播放逗留时间是在特定时间点查看媒体流所花费的时间，包括暂停、缓冲和开始时间。

通过“媒体播放逗留时间”面板，可以分析一段时间内的播放情况，并提供有关并发峰值以及划分和比较功能的详细信息。 要访问“媒体播放逗留时间”面板，请导航到启用了Media Analytics组件的报表包。 然后，单击最左侧的面板图标并将面板拖放到您的 Analysis Workspace 项目中。

此面板在日历中还包含新功能，让您能够选择和显示少于24小时的内容。 您可以在整个面板中执行此操作，也可以使用连续时间段创建区段，以便跟踪节目或节目部分的收视率导入/导出。 在放置了其中至少两个日期区段后，您将看到“日期序列显示”的单选按钮选项，该选项将覆盖具有相同x轴开始位置的行，或按其特定x轴开始位置的顺序显示这些行。

## 面板输入 {#Input}

您可以使用以下输入设置配置“媒体播放逗留时间”面板：

| 设置 | 描述 |
|---|---|
| 面板日期范围 | 面板日期范围的默认值为“今天”。您可以对其进行编辑以一次查看一天或几个月的数据。<br>这个可视化图表限制为 1440 行数据（例如，以分钟作为粒度级别来表示 24 小时）。如果日期范围和粒度的组合产生的行数超过了 1440 行，则将自动更新粒度以适应完整的日期范围。 |
| 粒度 | 粒度的默认值为“分钟”。<br>这个可视化图表限制为 1440 行数据（例如，以分钟作为粒度级别来表示 24 小时）。如果日期范围和粒度的组合产生的行数超过了 1440 行，则将自动更新粒度以适应完整的日期范围。 |
| 面板概要数字 | 要查看播放逗留时间的日期或时间详细信息，可使用概要数字。 “最大值”显示并发峰值的详细信息。“最小值”显示低谷期的详细信息。总计为所选内容的总播放时间。 面板默认仅显示“最大值”，但您可以将其更改为显示“最小值”、“总和”或这三者的任意组合。<br>如果您使用细分，则会为每一项显示概要数字。 |
| 系列细分 | （可选）您可以按区段、维度、维度项或日期范围细分可视化图表。<p>- 一次最多可以查看 10 行。细分仅限一个级别。</p><p>- 在拖动一个维度时，将根据所选面板日期范围，自动选择顶部维度项。</p>- 要比较日期范围，请将 2 个或更多日期范围拖动到系列细分过滤器中。 |
| 时间格式 | 您可以查看在“小时”:Minutes:“秒”（默认）或“分钟”（以整数显示，四舍五入到。5）中的播放时间。 |
| 日期序列显示 | 如果已将至少两个日期范围区段作为系列划分放置，您将看到选项，用于选择叠加（默认）或顺序。 叠加图将显示具有相同x轴开始位置的线条，以便它们并行运行，而顺序将显示具有其特定x轴开始位置的线条。 如果数据行向上（例如，区段1在晚上8:44结束，区段2在晚上8:45开始），则这些行将依次显示。 |

## 面板输出 {#Output}

“媒体播放逗留时间”面板会返回折线图和概要数字，以包含播放逗留时间的最大值、最小值和/或总和的详细信息。 在该面板顶部，提供了一个摘要行，用于提醒您选择的面板设置。

您可在任何时候，通过单击右上角的编辑铅笔来编辑并重建面板。

如果您选择了系列细分，则会为每一项在线形图上显示一条线以及概要数字：

![媒体播放逗留时间输出](assets/mpts_outputs1.png)

### 数据源

此面板中唯一可以使用的量度是播放逗留时间。

| 指标 | 描述 |
|---|---|
| 播放逗留时间 | 在选定粒度（包括暂停、缓冲和开始时间）内查看的内容的总小时数:minutes:秒（或分钟）。 |
