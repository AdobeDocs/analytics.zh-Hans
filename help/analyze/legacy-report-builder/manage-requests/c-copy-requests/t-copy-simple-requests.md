# 复制简单请求

{{legacy-arb}}

复制简单请求而不是引用请求。 简单请求不包含对其他请求或单元格内容的引用。

[引用请求](/help/analyze/legacy-report-builder/manage-requests/c-copy-requests/t-copy-referential-requests.md)使用单元格中的值作为参数的输入，如数据过滤器或关系过滤器。 这些筛选器使用匹配或趋势，并且基于先前请求的结果或基于用户输入的单元格（称为输入单元格）内容。

复制简单请求

1. 创建有效的请求。
1. 右键单击请求映射到的单元格之一，或者选择包含请求的单元格区域。

   在请求所涵盖的单元格组中选择要复制的单元格时，应保持一致。 首选选项是请求所涵盖的一组单元格中顶部和最左侧的单元格，并且从左到右依次工作。 这是因为Excel电子表格有数百个列和数千个行可用于向右和向下扩展。 如果您确实决定从与请求关联的一组单元格中最右边或最下面的单元格开始复制请求，并且要粘贴的单元格将超出电子表格的左边框或上边框，则系统将不允许您粘贴请求。
1. 选择&#x200B;**[!UICONTROL 复制请求]**。
1. 在电子表格的另一区域，右键单击一个空白单元格（不包含请求的单元格）。

   为了防止丢失或损坏已创建的请求，不能将包含请求的单元格粘贴到当前已映射请求的单元格。 如果复制或剪切包含请求的单元格，则右键单击包含请求的单元格（或单元格集）时，快捷菜单不会显示[!UICONTROL 粘贴请求]选项。 您必须选择其他单元格作为粘贴操作的目标，以便请求不会重叠。 无论您是选择带有待粘贴请求的单个单元格，还是包含请求的单元格区域，此参数均适用。
1. 单击&#x200B;**[!UICONTROL 粘贴请求]**。

   系统会根据原始请求的布置将原始请求的副本放置在一个或多个位置的单元格中。

   >[!NOTE]
   >
   >只复制请求，不复制单元格内容。 如果您有其他与了解单元格中显示的数据相关的信息（例如表列标题或行标识符），这些信息不是基于请求，而是基于的，请使用Excel的标准命令“复制”和“粘贴”。

   由于Excel使用不同的剪贴板来复制单元格内容和复制请求，因此您可以通过系列执行复制/粘贴和复制请求/粘贴请求，来复制非请求单元格内容和请求。 但是，如果您对电子表格中的请求应用格式，然后复制并粘贴，Report Builder将重现原始格式（例如边框、字体等） 在粘贴区域中。

   在粘贴请求之前修改已复制或剪切到剪贴板的请求会从剪贴板中删除请求。 因此，要使请求保持其原始状态，请不要在复制请求和粘贴请求之间修改请求。
