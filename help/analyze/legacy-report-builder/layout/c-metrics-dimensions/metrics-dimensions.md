# 量度和维度概述

{{legacy-arb}}

在“请求向导：第2步”中添加量度和维度，以定义请求中数据的Analytics布局。 Report Builder 中的量度和维度列表遵循 Reports and Analytics 中显示的菜单布局。

量度是数值。 它们包含有关您网站上活动的量化信息，如[!UICONTROL 页面查看次数]、[!UICONTROL 访客配置文件]、[!UICONTROL 点进次数]、[!UICONTROL 重新载入]、[!UICONTROL 平均逗留时间]、[!UICONTROL 日期]、[!UICONTROL 单位]等等。 Report Builder 将量度组织在多个组中，这些组遵循 Analytics 中的标准分组原则：

* 标准流量：根据指定的行项目显示指标。 例如，在页面报表中，“访问次数”量度显示对该特定页面的访问次数。
* 标准Commerce：收入、订单和结账等量度。
* 总量度：根据报表日期范围显示量度，而不考虑行项目。 此量度与其各自分析报表底部的总计相同。
* 参与率
* 已计算

维度是非数字值和日期，如性别、月份、年龄、忠诚度、监视器分辨率等。 维度是可以查看和比较的量度数据的描述或特性，例如在划分报表中。
