# Ad Hoc Analysis 使用入门{#concept_48936BA28FAE42DB81F1B2CD4726EB17}

>[!I重要信息]
>2018年8月6日，Adobe宣布计划终止临时分析。 确定终止日期后，Adobe 会立即与大家共享此信息。有关详细信息（包括在此期间兼容的 Java 版本），请访问 [https://adobe.ly/discoverworkspace](https://adobe.ly/discoverworkspace)。

您可以对网站分析执行即时、高级的活动。 您可以同时视图多个报表并跨多个维应用区段。 您可以从微观和宏观角度分析数据，以视图它们对重要业务指标的影响。

这些功能使您能够回答有关网站流量、访客人口统计、收入和产品移动的问题。 然后，您可以过滤、排序和细分数据，以找到精确问题的答案。 结果几乎即时返回，使您能够快速分析各种因素的影响。

## 系统建议 {#concept_6691331B45174290BD9B839806A9B52D}

尽管报表在一些最常用的 Web 浏览器上也能正常运行，但在符合特定建议要求的系统中运行时才能获得最佳的显示效果和功能。

<!-- 

c_sys_reqs.xml

 -->

>[!NOTE]从 2018 年 7 月起，Ad Hoc Analysis 将仅支持 Java 8 或更高版本。2018 年 7 月维护版本推出后，如果您选择在 Java 7 中运行 Ad Hoc Analysis，Adobe 将不再对您的 Ad Hoc Analysis 实施提供支持。

* 支持OpenGL 2.0的视频卡
* Cookie:必需
* 操作系统：Windows和Mac OS。
* Macromedia Flash Player:版本6或更高版本
* 显示器分辨率：800x600（建议1024x768）
* 颜色深度：16位或更高
* JavaScript:已启用
* Java版本：Java 1.7或更高版本（请参阅上面的说明）

   如果未安装正确版本的Java，则会为您安装该版本。 如果安装了不兼容版本的Java，临时分析会下载并提示您安装更新。

## Java升级说明 {#section_E4C0C6492FF24636A0FF71A59331111D}

从 2018 年 7 月起，Ad Hoc Analysis 将仅支持 Java 8 或更高版本。2018 年 7 月维护版本推出后，如果您选择在 Java 7 中运行 Ad Hoc Analysis，Adobe 将不再对您的 Ad Hoc Analysis 实施提供支持。

我们的。jar文件使用安全的256位加密进行签名，低于1.7.0_76的Java版本不支持该加密。 此256位证书使我们能够为您提供增强的安全性。

如果您安装的仍是 Java 7，则必须在 2018 年 7 月发布的维护版本之前进行升级。以下是具体操作方法：

* 如果允许您在计算机上安装项目:

   1. 转到 https://www.java.com。
   1. 单击 **[!UICONTROL Free Java Download]**.
   1. 单击 **[!UICONTROL Agree and Start Free Download]**.
   1. 安装特定于您的操作系统的最新Java版本。

* 如果不允 **许您** 在计算机上安装项目:

   1. 与您的IT部门合作，安装最新版Java。

## 从 Experience Cloud 登录 {#task_128ED319F3AE49ED886EA3DFA8D0987F}

<!--

1. 在浏览器中，导航至 [!DNL marketing.adobe.com]。
1. Type your company name, your username, and your password. Then click **[!UICONTROL Sign In]**.
1. 单击 **[!UICONTROL Adobe Analytics]** > **[!UICONTROL Tools]** > **[!UICONTROL Ad Hoc Analysis]**.

   如果此页面中未显示&#x200B;**启动 Ad Hoc Analysis** 按钮，请确保管理员已将您添加到管理工具的 *Ad hoc Analysis 授权用户*&#x200B;群组中。

1. 将文件 [!DNL discover.jnlp] 保存在本地。

   您可以随时运行该保存的文件来启动 Ad Hoc Analysis。

## 项目和工作区 {#concept_FAE346335B0347A192C6C806C775D72B}

项目定义导入的数据集，包括报表包和日期范围。 项目由任意数量的报告组成，其中包含所有指标、设置、维度和区段。 您可以开始新项目、加载已保存的项目或加载自动保存的项目。

<!-- 

c_projects.xml

 -->

将报告分组到工作 *区中*。 一个项目可以包含多个工作区，而一个工作区可以包含多个报告。 这些项目之间的关系最好理解为嵌套关系：

![](assets/project_workspace.png)

一次只能打开一个项目。 但是，您可以在项目中打开多个工作区。 在每个工作区中，您可以打开多个报告。

新项目的默认日期范围是 [!UICONTROL Last 90 Days]。

## 报表模板 {#concept_370F674C5B4C45368731AA801C5A45F8}

模板是您要执行的分析类型的起点。 模板可以是空白画布，如“排名”或“流失”报表。 或者，模板是一个报告，其中开始了默认量度和维度。

<!-- 

c_templates.xml

 -->

You can access templates when creating a project ( **[!UICONTROL File]** > **[!UICONTROL New Project]**), or by adding a workspace or report.

| 模板 | 描述 |
|--- |--- |
| 排名 | 提供一个空白画布，您可以在其中构建表。 例如，“页面”报表根据流量对您的网站上的页面进行排名，并且明细表会显示页面查看和收入等量度的百分比和数值。 |
| 趋势 | 允许您检查在报告期间内所选时间粒度（小时、日、周、月、季度或年）内的转化率和事件趋势。 |
| 总计 | 显示底线数字的行政级别报告。 它包含“总收入”、“页面视图”和“订单”的数据。 |
| 流失 | 允许您构建一个漏斗，它显示检查点之间的转换率和流失率。 例如，您可以在购买过程中跟踪访客的流失点。 |
| 流量 | 显示用户跨页面、站点部分和服务器访问的最常见路径。 |
| 转换漏斗 | 显示特定量度事件之间的转换百分比。 您可以使用此报告了解生成销售的点进次数和售出的件数。 |
| 网站分析 | 一个三维站点路径工具，它显示访客在指定页面和事件中的移动方式。 |
| 虚拟焦点组 | 从您的访问中随机访问，并向您显示大量有关访问的数据。 您可以使用报告根据访客群体创建焦点组。 |

## 打开报表 {#task_0AC455CDA198497AA546622FB05F300D}

您可以在创建项目时或从现有项目中打开报表或模板。 使用模板从头开始配置报表。

<!-- 

t_reports_opening.xml

 -->

有几种方法可用于打开报表：

* 在页面 [!UICONTROL New Report] 上，搜索报表或选择模板。
* 单 **[!UICONTROL Reports]** 击菜单，然后选择一个报告或报告模板。
* 从维启动报表：右键单击维名称，然后选择 **[!UICONTROL Run Report]** > **`report name`**。
