| 任务描述 | 说明和更多信息的链接 |
|--- |--- |
| **步骤1**：确保您的任何可能包含GDPR相关数据的报表包均映射到您的Experience Cloud(或IMS)组织。GDPR 请求是使用 Experience Cloud 组织提交的，并且将应用于由该组织声明的所有报表包。请求将不会应用于未映射到该组织的报表包，即使它们属于您的登录公司也是如此。 | 请参阅[将报表包映射到组织](https://docs.adobe.com/content/help/en/core-services/interface/about-core-services/report-suite-mapping.html)。 |
| **第步**&#x200B;设置数据保留策略。 | 为了使 Adobe 能够为 GDPR 数据访问/删除请求提供服务，需要设置数据保留策略。有关更多信息，请参阅 [Analytics 数据保留常见问题解答](/help/technotes/data-retention.md)。 |
| **步骤3**：使用DROE/GDPR标签、Adobe Analytics ID、命名空间和ID扩展使自己熟悉。 | 请阅读本文档集中的以下主题：<ul><li>[适用于 Analytics 变量的 GDPR 标签](/help/admin/c-data-governance/gdpr-labels.md)</li><li>[标签设置最佳实践](https://docs.adobe.com/content/help/en/analytics/admin/data-governance/gdpr-analytics-ids.html#concept_1BC4CA94B559481F8B08776DA100B23E)</li></ul> |
| **第步**：为报表套件中的每个变量分配标识、敏感度和数据管理标签。注意：请记住，每次创建新报表包后或在现有报表包中启用新变量时，都需要对标签设置进行审核。当启用新的解决方案集成时，您可能也需要审核标签设置，因为这些集成将出现要求设置标签的新变量。重新实施您的移动应用程序或网站时，可能会改变现有变量的使用方式，这或许也需要更新标签。 | 按照标签报告包数据中 [的说明](https://docs.adobe.com/content/help/en/analytics/admin/data-governance/gdpr-setup-reportsuite.html#concept_FAA948AD8CEA4BC38CB482EAF3648731)操作。 |
| **第步**：连接到Adobe GDPR API并提交访问和删除请求。 | 作为 Adobe Analytics 客户，您可以通过调用 [Adobe Experience Cloud GDPR API](https://www.adobe.io/apis/experienceplatform/gdpr.html)，提交单独的 GDPR 请求以访问和删除客户数据。您可以在请求中提交任何 Analytics 标识符（如[标签设置最佳实践](https://docs.adobe.com/content/help/en/analytics/admin/data-governance/gdpr-analytics-ids.html#concept_1BC4CA94B559481F8B08776DA100B23E)一节中所述）及其各自的命名空间 ID（数据源 ID）。 |
| **步骤6**：查看和管理报表包的GDPR设置。 | 按照 [查看报告包的“数据治理设置](/help/admin/c-data-governance/gdpr-view-settings.md)”中的说明操作。 |
